## 4. Voice Data Handling and Retention

### 4.1 Voice Command Processing
- **Audio Recording**: Your voice is captured only after wake word detection
- **Transcription**: Audio is immediately transcribed to text on our backend
- **Audio Deletion**: Original audio files are **deleted immediately** after transcription
- **Text Retention**: Transcribed text is retained for command history (up to 90 days)
- **Processing Location**: All voice processing occurs on secure backend servers in [specified region]
- **Encryption**: Voice data in transit is encrypted using TLS 1.3

### 4.2 Voice Model Training
- **No Personal Training**: Your voice is never used to train personalized voice models
- **Aggregated Training**: Only anonymized, aggregated transcription data may be used to improve general voice recognition
- **Opt-Out Available**: You can opt-out of analytics-based model improvement in Settings
- **Third-Party Services**: Voice responses use ElevenLabs text-to-speech service (no voice data shared, only intent text)

### 4.3 Voice History
- **User Access**: You can view your complete voice command history in the app
- **Deletion**: You can delete individual commands or your entire history at any time
- **Auto-Deletion**: Voice command history is automatically deleted after 90 days of inactivity

### 4.4 Sensitive Commands
- Commands containing passwords or credentials are **never** logged or stored
- Personal information mentioned in commands is not explicitly logged
- Only the intent and action are retained (not the specific values)

## 8. Security Measures

### 8.1 Data Protection
- **Encryption in Transit**: TLS 1.3 for all network communications
- **Encryption at Rest**: AES-256 encryption for sensitive data
- **Secure Storage**: iOS Keychain (FIPS-140 compliant) for OAuth tokens
- **Password Security**: Bcrypt hashing with salt for password storage
- **Session Management**: Secure session tokens with automatic expiration

### 8.2 Rate Limiting
- Login attempts: 5 per minute per IP address
- Registration: 3 per hour per IP address
- Password reset: 3 per hour per email
- General API: 100 requests per minute per user

### 8.3 Access Controls
- Role-based access control for internal systems
- Principle of least privilege for employee access
- Multi-factor authentication for admin accounts
- Regular security audits and penetration testing

### 8.4 Incident Response
