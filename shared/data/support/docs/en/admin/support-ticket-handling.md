# Support Ticket Handling

Manage customer support requests efficiently through the Bayit+ ticketing system. This guide covers ticket workflows, response procedures, and escalation paths.

## Ticket Queue Overview

Access support tickets from **Support** > **Tickets**. The queue displays:

- **New**: Unassigned incoming tickets
- **Open**: Assigned and in progress
- **Pending**: Awaiting customer response
- **Resolved**: Completed tickets

## Ticket Assignment

Tickets can be assigned manually or automatically:

**Auto-Assignment**: Tickets route based on category and agent availability. Configure rules in **Settings** > **Support** > **Routing**.

**Manual Assignment**: Supervisors can reassign tickets from the queue view.

## Working a Ticket

When handling a ticket:

1. Click the ticket to open the detail view
2. Review the customer's message and account history
3. Access their account details in the sidebar
4. Compose your response in the reply editor
5. Add internal notes visible only to staff
6. Update the ticket status appropriately

## Response Templates

Use templates for common issues:

| Template | Use Case |
|----------|----------|
| Password Reset | Account access issues |
| Billing Inquiry | Payment questions |
| Technical Support | Playback problems |
| Content Request | Missing content inquiries |

Access templates from the reply editor toolbar.

## Escalation Procedures

Escalate tickets requiring additional expertise:

1. Click **Escalate** in the ticket actions
2. Select the escalation reason
3. Choose the destination team or supervisor
4. Add context for the receiving agent
5. The ticket moves to the appropriate queue

## Service Level Agreements

Response time targets by priority:

- **Urgent**: First response within 1 hour
- **High**: First response within 4 hours
- **Normal**: First response within 24 hours
- **Low**: First response within 48 hours

Tickets approaching SLA deadlines are highlighted in the queue.

## Ticket Metrics

Track support performance:

- Average response time
- Resolution time by category
- Customer satisfaction scores
- Agent productivity statistics

Access metrics from **Support** > **Reports**.

## Closing Tickets

Before closing a ticket:

1. Confirm the issue is fully resolved
2. Request customer confirmation if appropriate
3. Select the resolution category
4. Add closing notes for future reference
5. Change status to **Resolved**
